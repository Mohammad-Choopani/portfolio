JavaScript Variables

A complete guide to using let, const, and var in modern JavaScript

🔸 let

The let keyword is used to declare variables that can be reassigned...

  • Can be updated but not re-declared in the same scope.
  • Exists only inside the block {} where it’s defined.
  • Does not hoist like var.
  • Commonly used in loops, conditions, and modern functions.
let age = 25;
age = 30; // ✅ Allowed
console.log(age);

🔸 const

The const keyword is used to declare variables that cannot be reassigned...

  • Cannot be updated or re-declared.
  • Safe for values you never want to accidentally change.
  • Still allows object and array mutation (careful!).
const name = "Mohammad";
console.log(name); // Output: Mohammad

🔸 var

The var keyword is the old way of declaring variables in JavaScript...

  • Can be re-declared and updated.
  • Is hoisted (can be used before it's declared).
  • Exists within functions (not blocks).
  • Replaced by let and const in ES6.
var country = "Canada";
country = "Iran";
console.log(country);
